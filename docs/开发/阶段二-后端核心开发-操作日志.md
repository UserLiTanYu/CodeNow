# 阶段二：后端核心开发 — 数据库与基础 CRUD — 操作日志

> 记录本阶段从建表到接口完成的全部操作步骤，供复盘复习使用。

---

## 步骤 1：创建建表 SQL（init.sql）

在 `codenow-backend/init.sql` 中编写 5 张表的建表语句，与开发文档第四节完全一致：

| 表名 | 说明 | 关键字段 |
|------|------|---------|
| `sys_user` | 用户表 | username、password（BCrypt）、nickname、role |
| `blog_article` | 文章表 | title、content、category_id、author_id、status |
| `blog_category` | 分类表 | name、description、sort |
| `blog_tag` | 标签表 | name |
| `blog_article_tag` | 文章-标签关联表 | article_id、tag_id |

- 使用 `utf8mb4` 字符集、`InnoDB` 引擎
- 插入一条默认管理员数据（admin / 123456，BCrypt 加密）
- 在 MySQL 中执行 `init.sql`，确认 5 张表创建成功

---

## 步骤 2：创建公共类（common 包）

在 `com.codenow.common` 下创建：

- **R.java**：统一返回格式 `{code, message, data}`，提供 `ok()` / `error()` 静态方法
- **PageQuery.java**：分页查询参数（pageNum 默认 1，pageSize 默认 10）

---

## 步骤 3：创建实体类（entity 包）

在 `com.codenow.entity` 下为 5 张表各创建一个实体类：

- 使用 `@Data`（Lombok）自动生成 getter/setter
- 使用 `@TableName` 绑定数据库表名
- 使用 `@TableId(type = IdType.AUTO)` 标注主键自增
- 使用 `@TableLogic` 标注逻辑删除字段（isDeleted）
- 字段类型与数据库一一对应：BIGINT → Long，VARCHAR → String，TEXT → String，TINYINT → Integer，DATETIME → LocalDateTime

---

## 步骤 4：创建 Mapper 接口（mapper 包）

在 `com.codenow.mapper` 下创建 5 个 Mapper 接口，均继承 `BaseMapper<T>` 并添加 `@Mapper` 注解：

- `SysUserMapper`
- `BlogArticleMapper`
- `BlogCategoryMapper`
- `BlogTagMapper`
- `BlogArticleTagMapper`

MyBatis-Plus 的 `BaseMapper` 已内置 selectById、insert、updateById、deleteById 等通用方法，无需手写 SQL。

---

## 步骤 5：创建 Service 层（service 包）

在 `com.codenow.service` 下创建 5 个 Service 接口，均继承 `IService<T>`。

在 `com.codenow.service.impl` 下创建 5 个实现类，继承 `ServiceImpl<Mapper, Entity>` 并添加 `@Service` 注解：

| 接口 | 实现类 |
|------|--------|
| `SysUserService` | `SysUserServiceImpl` |
| `BlogArticleService` | `BlogArticleServiceImpl` |
| `BlogCategoryService` | `BlogCategoryServiceImpl` |
| `BlogTagService` | `BlogTagServiceImpl` |
| `BlogArticleTagService` | `BlogArticleTagServiceImpl` |

`IService` 已内置 save、removeById、updateById、getById、list、page 等通用业务方法。

---

## 步骤 6：创建 DTO（dto 包）

在 `com.codenow.dto` 下创建 4 个数据传输对象：

- **LoginDTO**：登录请求（username、password），使用 `@NotBlank` 校验
- **ArticleDTO**：文章创建/更新请求（title、content、summary、categoryId、status、tagIds）
- **CategoryDTO**：分类创建/更新请求（name、description、sort）
- **TagDTO**：标签创建/更新请求（name）

DTO 用于接收前端传参，与实体类解耦，避免直接暴露数据库字段。

---

## 步骤 7：创建全局异常处理（exception 包）

在 `com.codenow.exception` 下创建：

- **BusinessException.java**：自定义业务异常，继承 RuntimeException，携带 code 和 message
- **GlobalExceptionHandler.java**：`@RestControllerAdvice` 全局异常处理器，捕获以下异常并统一返回 R 格式：
  - `BusinessException` → 返回对应 code 和 message
  - `MethodArgumentNotValidException` → 返回 400 + 首个校验错误信息
  - `Exception` → 返回 500 + "系统异常"

---

## 步骤 8：创建 Controller 层（controller 包）

在 `com.codenow.controller` 下创建 4 个控制器，使用 `@Tag`（Knife4j）标注模块名称：

**AuthController**（`/api/auth`）：
- `POST /login` — 校验用户名密码，Sa-Token 创建会话，返回 Token
- `POST /logout` — Sa-Token 注销会话
- `GET /me` — 根据 Token 获取当前用户信息（密码置空）

**ArticleController**（`/api/articles`）：
- `GET /` — 分页查询文章列表
- `GET /{id}` — 查询单篇文章详情
- `POST /` — 新增文章
- `PUT /{id}` — 修改文章
- `DELETE /{id}` — 删除文章（逻辑删除）

**CategoryController**（`/api/categories`）：
- `GET /` — 查询分类列表（按 sort 排序）
- `POST /` — 新增分类
- `PUT /{id}` — 修改分类
- `DELETE /{id}` — 删除分类

**TagController**（`/api/tags`）：
- `GET /` — 查询标签列表
- `POST /` — 新增标签
- `DELETE /{id}` — 删除标签

---

## 步骤 9：创建配置类（config 包）

在 `com.codenow.config` 下创建 4 个配置类：

- **SecurityConfig**：禁用 Spring Security 默认拦截（仅使用其 BCrypt 加密功能），认证由 Sa-Token 负责
- **SaTokenConfig**：注册 Sa-Token 拦截器，对 `/api/auth/login` 放行，其余 `/api/**` 需登录
- **CorsConfig**：配置 CORS 跨域过滤器，允许所有来源访问
- **MybatisPlusConfig**：注册分页插件 `PaginationInnerInterceptor`，支持 MySQL 分页查询

---

## 步骤 10：修复依赖兼容性问题

开发过程中遇到并修复了以下问题：

| 问题 | 原因 | 修复方式 |
|------|------|---------|
| Mapper Bean 未注册 | `@Mapper` 注解未被扫描 | 启动类添加 `@MapperScan("com.codenow.mapper")` |
| Spring Boot 4.0.7 启动报 sqlSessionFactory 缺失 | MyBatis-Plus 3.5.7 不兼容 Spring Boot 4.x | 降级 Spring Boot 到 3.4.7 |
| starter-webmvc 找不到 | Spring Boot 3.x 中名称不同 | 改为 `spring-boot-starter-web` |
| starter-webmvc-test 找不到 | Spring Boot 3.x 中名称不同 | 改为 `spring-boot-starter-test` |
| Bean Validation 提示缺失 | 缺少校验提供者依赖 | 添加 `spring-boot-starter-validation` |
| Sa-Token 连接 Redis 失败 | 本地未安装 Redis | 移除 Redis 依赖，Sa-Token 改用内存存储 |
