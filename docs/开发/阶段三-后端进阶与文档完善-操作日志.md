| 文件 | 补充的注解 |
|------|-----------|
| `LoginDTO.java` | 类级 `@Schema(description="登录请求参数")`，username/password 字段加 `@Schema` 含 description 和 example |
| `ArticleDTO.java` | 类级 `@Schema`，7 个字段全部加 `@Schema`（title、content、summary、coverImage、categoryId、status、tagIds） |
| `CategoryDTO.java` | 类级 `@Schema`，name/description/sort 字段加 `@Schema` |
| `TagDTO.java` | 类级 `@Schema`，name 字段加 `@Schema` |
| `PageQuery.java` | 类级 `@Schema(description="分页查询参数")`，pageNum/pageSize 字段加 `@Schema` |

## 步骤 3：修复 Knife4j 文档页不显示接口的问题

发现 `http://localhost:8080/doc.html` 页面可以打开但接口列表为空，排查并修复两层原因：

**原因一：application.yaml 的 group-configs 配置有误**

`paths-to-match: /**` 在 YAML 中被解析为非预期值，导致 springdoc 没有扫描任何路径。

修复：去掉 `group-configs`，简化为 `packages-to-scan: com.codenow.controller`，让 springdoc 自动发现 Controller。

**原因二：springdoc-openapi 版本不兼容**

Knife4j 4.5.0 自带的 springdoc-openapi 2.3.0 与 Spring Boot 3.4.7（Spring Framework 6.2.8）不兼容，报 `NoSuchMethodError: ControllerAdviceBean`。

修复：在 pom.xml 中显式添加 springdoc-openapi 2.8.6 依赖，覆盖 Knife4j 自带的旧版本。

**原因三：Sa-Token 拦截器排除路径不完整**

补充 `/swagger-ui/**` 到 SaTokenConfig 的排除路径列表。

## 步骤 4：创建 ArticleVO 实现文章联表查询

新建 `ArticleVO.java`，封装文章详情的完整信息：

```java
public class ArticleVO {
    private BlogArticle article;   // 文章基本信息
    private String categoryName;   // 分类名称
    private List<BlogTag> tags;    // 标签列表
}
```

设计思路：采用组装模式而非复杂 SQL 联表查询——先查文章，再分别查分类和标签，代码更清晰易维护。

---

## 步骤 5：实现文章创建/修改时关联标签

改造 `BlogArticleService`，新增以下方法：

- `saveArticleWithTags(article, tagIds)`：保存文章后，遍历 tagIds 插入 `blog_article_tag` 中间表
- `updateArticleWithTags(article, tagIds)`：更新文章后，先删除旧关联再插入新关联
- `deleteArticleWithTags(id)`：删除文章时，同步清理中间表记录

使用 `@Transactional` 注解保证操作原子性。

---

## 步骤 6：实现文章列表按分类/标签筛选

改造 `pageArticleVO` 方法，支持两个可选筛选参数：

- `categoryId`：通过 `LambdaQueryWrapper.eq()` 条件直接过滤
- `tagId`：先查 `blog_article_tag` 中间表拿到文章 ID 列表，再用 `IN` 条件过滤

两个参数可单独使用，也可组合使用。

---

## 步骤 7：新增文章状态切换接口

在 `ArticleController` 中新增接口：

```
PUT /api/articles/{id}/status
```

逻辑：查询文章当前 status，0 切换为 1（草稿→发布），1 切换为 0（发布→草稿）。

---

## 步骤 8：改造 ArticleController 使用新 Service 方法

对 ArticleController 的 5 个接口进行改造：

| 接口 | 改造内容 |
|------|---------|
| `GET /api/articles` | 返回类型从 `Page<BlogArticle>` 改为 `Page<ArticleVO>`，新增 categoryId/tagId 筛选参数 |
| `GET /api/articles/{id}` | 返回类型改为 `ArticleVO`，调用 `getArticleVOById` |
| `POST /api/articles` | 调用 `saveArticleWithTags` 替代 `save`，支持传入 tagIds |
| `PUT /api/articles/{id}` | 调用 `updateArticleWithTags` 替代 `updateById` |
| `DELETE /api/articles/{id}` | 调用 `deleteArticleWithTags` 替代 `removeById` |
