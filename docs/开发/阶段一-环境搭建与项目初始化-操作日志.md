# 阶段一：环境搭建与项目初始化 — 操作日志

> 记录本阶段从零开始搭建项目的全部操作步骤，供复盘复习使用。

---

## 步骤 1：安装开发环境

依次安装以下工具并验证版本：

| 工具 | 安装版本 | 验证命令 | 说明 |
|------|---------|---------|------|
| JDK | 21.0.11 | `java -version` | 最初安装了 JDK 25，因兼容性问题改为 JDK 21 |
| Maven | 3.9.4 | `mvn -version` | 确认 Maven 识别的 Java 版本为 21 |
| Node.js | v24.14.1 | `node -v` | 同时附带 npm 11.11.0 |
| MySQL | 9.6.0 | `mysql --version` | 比推荐的 8.0 新，向下兼容可用 |
| Redis | 3.2.100 | 手动启动 `redis-server.exe` | 版本较老但够用，需手动启动服务 |

---

## 步骤 2：在 IDEA 中创建后端项目

- 打开 IDEA → File → New → Project → Spring Initializr
- 配置：Name = `codenow-backend`，Group = `com.codenow`，JDK = 21，Type = Maven
- 勾选依赖：Spring Web、MySQL Driver、MyBatis Framework、Lombok
- 点击 Create 生成项目，项目路径：`D:\F\codenow-backend\codenow-backend`

---

## 步骤 3：配置后端 pom.xml

在 `pom.xml` 中添加以下依赖（替换原生 MyBatis）：

- **MyBatis-Plus 3.5.7**：增强版 ORM 框架，内置通用 CRUD
- **Sa-Token 1.39.0**：轻量级权限认证框架（含 Redis 整合）
- **Knife4j 4.5.0**：API 文档自动生成工具
- **commons-pool2**：Redis 连接池

---

## 步骤 4：配置后端 application.yaml

修改 `src/main/resources/application.yaml`，配置以下内容：

- 服务器端口：8080
- MySQL 数据源：`jdbc:mysql://localhost:3306/codenow`，用户名 root，密码 123456
- Redis：localhost:6379
- MyBatis-Plus：驼峰映射、逻辑删除、分页插件、SQL 日志
- Sa-Token：Token 名称、有效期、UUID 风格
- Knife4j：API 文档路径配置

---

## 步骤 5：创建 MySQL 数据库

在终端或 Navicat 中执行：
```sql
CREATE DATABASE codenow DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_general_ci;
```

---

## 步骤 6：验证后端项目启动

在 IDEA 中运行 `CodenowBackendApplication`，确认看到：
- `Tomcat started on port 8080 (http)`
- `Sa-Token v1.39.0` 启动日志
- `Started CodenowBackendApplication in x.x seconds`

---

## 步骤 7：创建前端 Vue 3 项目

在终端执行：
```bash
cd c:\Users\litan\Desktop\code\新项目
npm create vue@latest codenow-frontend -- --router --pinia --eslint
```

选项：启用 Router、Pinia、ESLint，不启用 TypeScript 和 JSX。

---

## 步骤 8：安装前端依赖

```bash
cd codenow-frontend
npm install
npm install element-plus axios md-editor-v3
```

- **Element Plus**：Vue 3 UI 组件库
- **Axios**：HTTP 请求库
- **md-editor-v3**：Markdown 编辑器组件

---

## 步骤 9：验证前端项目启动

执行 `npm run dev`，确认看到：
```
VITE v8.1.4 ready in 1669 ms
Local: http://localhost:5173/
```

---

## 步骤 10：初始化 Git 仓库

- 在项目根目录 `c:\Users\litan\Desktop\code\新项目\` 执行 `git init`
- 创建 `.gitignore` 文件，同时覆盖前后端项目的忽略规则：
  - 后端：target/、.idea/、*.iml、mvnw、mvnw.cmd
  - 前端：node_modules/、dist/
  - 通用：*.log、.DS_Store、.env 等
- 执行 `git add -A` 和 `git commit`，提交前端项目的初始 commit

---

## 步骤 11：将后端项目纳入 Git 管理

- 后端项目最初创建在 `D:\F\codenow-backend\codenow-backend\`，不在 Git 仓库内
- 将后端项目复制到项目根目录下：`cp -r D:\F\codenow-backend\codenow-backend codenow-backend`
- 确认 .gitignore 已正确忽略后端的 target/ 目录（编译产物不提交）
- 执行 `git add codenow-backend/` 和 `git commit`，提交后端项目
