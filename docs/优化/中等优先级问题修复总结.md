# 中等优先级问题修复总结

> 执行日期：2026-07-15
> 依据：`项目系统性评估报告.md` 全文中显式标记为“中”的条目，以及第五章“中优先级”汇总条目。去重后共 23 项。

## 1. 问题提取、分类与影响

| # | 分类 | 问题 | 影响与风险 |
|---|---|---|---|
| 1 | 稳定性 | `AuthController.me()` 未检查空用户 | 已删除用户持有旧 Token 时触发 NPE |
| 2 | 数据一致性 | 状态/置顶切换存在 TOCTOU | 并发请求可能丢失切换结果 |
| 3 | 数据一致性 | 评论未校验 `articleId` | 产生指向不存在文章的孤立评论 |
| 4 | 可用性 | 异步任务未配置线程池 | 高并发下线程无上限，可能耗尽资源 |
| 5 | 数据一致性 | 递归删除评论缺少原子性 | 中途失败可能只删除部分评论 |
| 6 | 数据一致性 | 更新文章未校验标签 ID | 产生无效关联或数据库约束异常 |
| 7 | 性能 | `highlight.js` 全量打包 | 文章页 JavaScript 体积和加载时间增加 |
| 8 | 性能 | Element Plus CSS 全量导入 | 首屏样式资源冗余 |
| 9 | 部署安全 | 生产 MySQL 禁用 TLS | 数据库认证和业务数据可能明文传输 |
| 10 | 部署安全 | MySQL/Redis 暴露宿主机端口 | 扩大数据库与缓存攻击面 |
| 11 | 质量门禁 | Docker 构建跳过测试 | 失败代码可能被打入生产镜像 |
| 12 | 用户体验 | 博客加载失败静默处理 | 用户无法判断是空数据还是网络故障 |
| 13 | 用户体验 | 编辑文章离开无确认 | 未保存内容可能误丢失 |
| 14 | 功能完整性 | 标签缺少编辑入口 | 无法在后台完成标签重命名闭环 |
| 15 | 代码规范 | 客户端 IP 获取逻辑重复 | 规则不一致、后续安全修复容易遗漏 |
| 16 | 代码规范 | 状态与根评论魔法数字散落 | 语义不清，修改时容易不一致 |
| 17 | 性能 | 评论树无分页 | 大文章一次加载全部评论，增加内存和响应耗时 |
| 18 | 数据一致性 | 文章软删而标签关联硬删 | 恢复文章后标签永久丢失 |
| 19 | 认证安全 | UUID Token 无签名 | Token 本身不能做签名完整性校验 |
| 20 | API 规范 | 全局异常始终返回 HTTP 200 | 代理、监控和客户端无法按 HTTP 语义处理错误 |
| 21 | 用户体验 | 评论数只统计根评论 | 页面显示数量与实际讨论数不一致 |
| 22 | 用户体验 | 图片上传无客户端类型/大小校验 | 无效文件消耗上传带宽，反馈滞后 |
| 23 | 用户体验 | 浏览器标题固定 | 多标签页难以辨识当前页面 |

## 4. 部署注意事项与遗留项

- 已有数据库必须在部署新后端前执行 `codenow-backend/migration-medium-priority.sql`；全新数据库无需执行。
- `JWT_SECRET` 变更会使旧 UUID Token 全部失效，升级后管理员需重新登录，这是切换签名 Token 的预期行为。
- `DB_SSL_MODE` 默认 `REQUIRED` 只保证加密。具备受信 CA 的生产数据库建议设置为 `VERIFY_IDENTITY`，并配置客户端信任证书。
- 评论分页按“根评论”分页，一个根评论下的全部回复随该页加载；这保留完整对话上下文，但极端单楼回复量仍可能较大。若出现单楼数万回复，应进一步对子回复做游标分页。
