# 码上记（CodeNow）个人博客系统 — 开发文档

> 文档版本：v1.0 | 预估开发周期：2-3 周（每天 3-4 小时）

---

## 一、项目概述

| 项目 | 内容 |
|------|------|
| **项目名称** | 码上记 / CodeNow |
| **一句话简介** | 一个支持 Markdown 写作、文章分类与标签管理的个人技术博客，帮助开发者记录和分享学习笔记。 |
| **目标用户** | 技术爱好者、个人开发者、正在学习编程的大三学生 |
| **项目定位** | 前后端分离的全栈练手项目，可部署上线作为求职作品集展示 |

---

## 二、技术选型与理由

### 后端技术栈

| 技术 | 版本 | 选择理由（通俗版） |
|------|------|-------------------|
| **JDK** | 17（LTS） | Java 17 是长期支持版本，稳定且企业主流，Spring Boot 3.x 最低要求 JDK 17 |
| **Spring Boot** | 3.2+ | Java Web 开发的"万能脚手架"，帮你自动配置好 Tomcat、数据库连接等，省去大量样板代码 |
| **MyBatis-Plus** | 3.5+ | MyBatis 的增强版，内置通用 CRUD 方法，写 SQL 更少，分页查询一行代码搞定 |
| **MySQL** | 8.0 | 最流行的关系型数据库，免费、资料多、面试必问，适合博客这类结构化数据 |
| **Redis** | 7.x | 内存数据库，速度极快。用来缓存热门文章、存储登录 Token，减轻 MySQL 压力 |
| **Sa-Token** | 1.38+ | 轻量级权限认证框架，比 Spring Security 简单 10 倍，适合新手快速实现登录和权限控制 |
| **Knife4j** | 4.x | Swagger 的美化增强版，自动生成 API 文档页面，方便前后端联调 |
| **Lombok** | 最新 | 自动生成 getter/setter/构造方法，减少大量重复代码，让实体类更简洁 |
| **MapStruct** | 最新 | 对象转换工具，把数据库实体（DO）转成前端需要的 DTO，类型安全且性能好 |

### 前端技术栈

| 技术 | 版本 | 选择理由（通俗版） |
|------|------|-------------------|
| **Vue 3** | 3.4+ | 目前最火的前端框架之一，组合式 API 写法更灵活，国内生态好、中文资料多 |
| **Vite** | 5.x | Vue 项目的构建工具，比 Webpack 快 10 倍以上，开发时改代码页面秒刷新 |
| **Element Plus** | 2.x | 基于 Vue 3 的 UI 组件库，表格、表单、弹窗等开箱即用，不用从零写样式 |
| **Vue Router** | 4.x | 前端路由管理，实现页面跳转（如首页→文章详情）无需刷新整个页面 |
| **Pinia** | 2.x | Vue 3 推荐的状态管理工具，比 Vuex 更简洁，用于管理登录状态、用户信息等全局数据 |
| **Axios** | 1.x | HTTP 请求库，用于前端调用后端 API，支持拦截器（如自动带上 Token） |
| **md-editor-v3** | 最新 | Vue 3 的 Markdown 编辑器组件，支持实时预览、代码高亮，开箱即用 |

### 开发与部署工具

| 工具 | 用途 |
|------|------|
| **IDEA** | Java 后端开发 IDE |
| **VSCode** | 前端开发 IDE + Volar 插件（Vue 3 支持） |
| **Git** | 版本管理，托管到 Gitee/GitHub |
| **Navicat / DBeaver** | 数据库可视化管理工具 |
| **Postman** | API 接口测试工具 |

| **Nginx** | 反向代理，将域名请求转发到前端和后端服务 |

## 三、功能模块规划

### 3.1 核心功能（必需，优先开发）

| 模块 | 功能点 | 对应页面 | 对应 API |
|------|--------|---------|---------|
| **用户模块** | 登录 / 登出 / 获取当前用户信息 | 登录页 | `POST /api/auth/login`、`POST /api/auth/logout`、`GET /api/auth/me` |
| **文章模块** | 文章 CRUD（创建、查询、修改、删除） | 文章列表页、文章详情页、文章编辑页 | `GET /api/articles`（分页）、`GET /api/articles/{id}`、`POST /api/articles`、`PUT /api/articles/{id}`、`DELETE /api/articles/{id}` |
| **文章模块** | 文章草稿与发布状态切换 | 文章编辑页 | `PUT /api/articles/{id}/status` |
| **分类模块** | 分类 CRUD | 分类管理页 | `GET /api/categories`、`POST /api/categories`、`PUT /api/categories/{id}`、`DELETE /api/categories/{id}` |
| **标签模块** | 标签 CRUD，文章关联标签 | 文章编辑页（标签选择） | `GET /api/tags`、`POST /api/tags`、`DELETE /api/tags/{id}` |
| **Markdown** | Markdown 编辑与实时预览 | 文章编辑页 | 前端组件，不涉及后端 API |
| **分页** | 文章列表分页查询 | 文章列表页 | `GET /api/articles?page=1&size=10` |

### 3.2 可选功能（按优先级排序，时间充裕再做）

| 优先级 | 模块 | 功能点 | 说明 |
|--------|------|--------|------|
| P1 | **文章置顶/推荐** | 管理员可置顶文章 | 增加一个 `is_top` 字段即可，改动小、效果明显 |
| P1 | **文章浏览量** | 记录每篇文章的阅读次数 | 简单计数器，可用 Redis 缓存定期落库 |
| P2 | **评论系统** | 文章下方可评论，支持回复 | 需要新建评论表，涉及父子评论嵌套查询 |
| P2 | **全文搜索** | 按关键词搜索文章标题和内容 | 可用 MySQL LIKE 简单实现，进阶可用 Elasticsearch |
| P3 | **文章归档** | 按月份归档文章 | 按创建时间分组统计，纯展示功能 |
| P3 | **访客统计** | 首页展示总文章数、总访问量等 | 简单的数据统计接口 |

## 四、数据库设计

### 4.1 用户表（`sys_user`）

|--------|------|------|------|

| `username` | VARCHAR(50) | UNIQUE, NOT NULL | 用户名（登录账号） |
| `password` | VARCHAR(100) | NOT NULL | 密码（BCrypt 加密存储） |
| `nickname` | VARCHAR(50) | | 昵称（展示用） |
| `avatar` | VARCHAR(255) | | 头像 URL |
| `email` | VARCHAR(100) | | 邮箱 |
| `role` | VARCHAR(20) | DEFAULT 'admin' | 角色（admin/user） |
| `create_time` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `update_time` | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |
| `is_deleted` | TINYINT | DEFAULT 0 | 逻辑删除（0=正常, 1=已删） |

### 4.2 文章表（`blog_article`）

| 字段名 | 类型 | 约束 | 备注 |

| `cover_image` | VARCHAR(255) | | 封面图 URL |
| `category_id` | BIGINT | FOREIGN KEY | 所属分类 ID |
| `author_id` | BIGINT | FOREIGN KEY | 作者 ID |
| `status` | TINYINT | DEFAULT 0 | 状态（0=草稿, 1=已发布） |
| `is_top` | TINYINT | DEFAULT 0 | 是否置顶（0=否, 1=是） |
| `view_count` | INT | DEFAULT 0 | 浏览量 |
| `create_time` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `update_time` | DATETIME | ON UPDATE CURRENT_TIMESTAMP | 更新时间 |
| `is_deleted` | TINYINT | DEFAULT 0 | 逻辑删除 |

### 4.3 分类表（`blog_category`）

| 字段名 | 类型 | 约束 | 备注 |
|--------|------|------|------|
| `id` | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 主键 |
| `name` | VARCHAR(50) | UNIQUE, NOT NULL | 分类名称 |

| `sort` | INT | DEFAULT 0 | 排序号（数字越小越靠前） |

| `is_deleted` | TINYINT | DEFAULT 0 | 逻辑删除 |

### 4.4 标签表（`blog_tag`）

| 字段名 | 类型 | 约束 | 备注 |
|--------|------|------|------|
| `id` | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 主键 |
| `name` | VARCHAR(50) | UNIQUE, NOT NULL | 标签名称 |
| `create_time` | DATETIME | DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| `is_deleted` | TINYINT | DEFAULT 0 | 逻辑删除 |

### 4.5 文章-标签关联表（`blog_article_tag`）

| 字段名 | 类型 | 约束 | 备注 |
|--------|------|------|------|
| `id` | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 主键 |
| `article_id` | BIGINT | FOREIGN KEY, NOT NULL | 文章 ID |
| `tag_id` | BIGINT | FOREIGN KEY, NOT NULL | 标签 ID |

### 4.6 表间关系说明

```
用户(1) ──< 文章(N)        一个用户（博主）写多篇文章
分类(1) ──< 文章(N)        一个分类下有多篇文章
文章(N) >──< 标签(N)       多对多，通过 article_tag 中间表关联
```

- `blog_article.category_id` → 关联 `blog_category.id`（多对一）
- `blog_article.author_id` → 关联 `sys_user.id`（多对一）
- `blog_article_tag.article_id` → 关联 `blog_article.id`
- `blog_article_tag.tag_id` → 关联 `blog_tag.id`

## 五、开发流程与阶段划分

> 总周期：2-3 周，按"后端先行、前端跟进、最后联调部署"的顺序推进。

---

### 第一阶段：环境搭建与项目初始化（第 1-2 天）

**任务目标：** 搭建好开发环境，跑通前后端空项目

| 任务 | 产出物 |
|------|--------|
| 安装 JDK 17、Maven、Node.js 18+、MySQL 8.0、Redis | 本地环境可运行 |
| 使用 Spring Initializr 创建后端项目（勾选 Web、MyBatis、Lombok 等依赖） | `codenow-backend` 项目骨架 |
| 使用 `npm create vue@latest` 创建前端项目 | `codenow-frontend` 项目骨架 |
| 初始化 Git 仓库，创建 `dev` 分支 | 远程仓库（Gitee/GitHub） |
| 配置 application.yml（数据库连接、端口号等） | 后端启动不报错 |
| 安装前端依赖（Element Plus、Axios、Vue Router、Pinia） | 前端 `npm run dev` 能打开 |

**验收标准：** 后端访问 `http://localhost:8080` 返回默认响应；前端访问 `http://localhost:5173` 看到 Vue 默认页面。

---

**任务目标：** 完成数据库建表 + 用户登录 + 文章/分类/标签的增删改查接口

| 任务 | 产出物 |

| 实现文章 CRUD 全部 5 个接口 | 文章增删改查 + 分页 |
| 实现分类 CRUD 全部 4 个接口 | 分类增删改查 |

## 六、部署与展示方案

### 6.1 服务器准备

**推荐配置：**
- 阿里云 / 腾讯云 ECS（2核4G 即可，新用户有优惠，约 99 元/年）
- 操作系统：Ubuntu 22.04 LTS 或 CentOS Stream 9
- 开放端口：80（HTTP）、443（HTTPS）、22（SSH）

**购买后第一步 — 连接服务器：**
```bash
ssh root@你的服务器IP
```

### 6.2 安装 Docker 环境

```bash
# Ubuntu 示例
apt update && apt install -y docker.io docker-compose
systemctl enable docker && systemctl start docker
docker --version  # 确认安装成功
```

### 6.3 编写 docker-compose.yml

在服务器创建目录 `/opt/codenow/`，上传以下文件：

**docker-compose.yml：**
```yaml
version: '3.8'
services:
  mysql:
    image: mysql:8.0
    container_name: codenow-mysql
    environment:
      MYSQL_ROOT_PASSWORD: your_password
      MYSQL_DATABASE: codenow
    ports:
      - "3306:3306"
    volumes:
      - mysql_data:/var/lib/mysql
      - ./init.sql:/docker-entrypoint-initdb.d/init.sql
    restart: always

  redis:
    image: redis:7-alpine
    container_name: codenow-redis
    ports:
      - "6379:6379"
    restart: always

  backend:
    image: openjdk:17-slim
    container_name: codenow-backend
    working_dir: /app
    volumes:
      - ./codenow-backend.jar:/app/app.jar
    command: java -jar app.jar --spring.profiles.active=prod
    ports:
      - "8080:8080"
    depends_on:
      - mysql
      - redis
    restart: always

  frontend:
    image: nginx:alpine
    container_name: codenow-frontend
    volumes:
      - ./dist:/usr/share/nginx/html
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
    ports:
      - "80:80"
    depends_on:
      - backend
    restart: always

volumes:
  mysql_data:
```

**nginx.conf（放在 /opt/codenow/ 下）：**
```nginx
server {
    listen 80;
    server_name 你的域名或IP;

    # 前端静态文件
    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;  # Vue 路由 history 模式支持
    }

    # 后端 API 代理
    location /api/ {
        proxy_pass http://backend:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
```

### 6.4 部署步骤

```bash
# 1. 本地打包后端
cd codenow-backend
mvn clean package -DskipTests
# 生成 target/codenow-backend.jar

# 2. 本地打包前端
cd codenow-frontend
npm run build
# 生成 dist/ 目录

# 3. 上传到服务器（任选一种方式）
# 方式 A：scp 命令
scp target/codenow-backend.jar root@服务器IP:/opt/codenow/
scp -r dist/* root@服务器IP:/opt/codenow/dist/

# 方式 B：使用 FileZilla 等 FTP 工具拖拽上传

# 4. 同时上传 init.sql、nginx.conf、docker-compose.yml 到 /opt/codenow/

# 5. 启动服务
cd /opt/codenow
docker-compose up -d

# 6. 查看运行状态
docker-compose ps
# 所有服务状态应为 Up
```

### 6.5 域名配置（可选但推荐）

| 步骤 | 操作 |
|------|------|
| 1 | 在阿里云/腾讯云购买域名（如 `codenow.cn`，约 29 元/年） |
| 2 | 在域名控制台添加 A 记录，主机记录 `@`，记录值填服务器 IP |
| 3 | 等待 DNS 解析生效（通常 10 分钟内） |
| 4 | 申请免费 SSL 证书（Let's Encrypt） |
| 5 | 修改 nginx.conf 的 `listen` 为 `443 ssl` 并配置证书路径 |
| 6 | 访问 `https://你的域名` 即可 |

### 6.6 最终访问地址

| 场景 | 访问地址 |
|------|---------|
| 无域名 | `http://服务器IP` |
| 有域名（HTTP） | `http://你的域名` |
| 有域名（HTTPS） | `https://你的域名` |
| 后端 API 文档 | `http://服务器IP/doc.html` |
