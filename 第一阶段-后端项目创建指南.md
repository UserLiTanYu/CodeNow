# 第一阶段：后端 Spring Boot 项目创建指南

## 一、在 IDEA 中创建项目

### 步骤 1：打开项目创建向导

打开 IntelliJ IDEA → **File** → **New** → **Project**

### 步骤 2：选择 Spring Initializr

在左侧列表中选择 **Spring Initializr**，右侧配置如下：

| 配置项 | 填写内容 |
|--------|---------|
| Name | `codenow-backend` |
| Location | `c:\Users\litan\Desktop\code\新项目\codenow-backend` |
| Language | Java |
| Type | Maven |
| Group | `com.codenow` |
| Artifact | `codenow-backend` |
| Package name | `com.codenow` |
| JDK | 21 |
| Java | 21 |
| Packaging | Jar |

### 步骤 3：选择依赖

点击 **Next**，在依赖勾选页面搜索并勾选以下依赖：

| 分类 | 依赖名称 | 说明 |
|------|---------|------|
| Web | Spring Web | 提供 Web 开发基础能力 |
| SQL | MySQL Driver | MySQL 数据库驱动 |
| SQL | MyBatis Framework | ORM 框架（后续手动升级为 MyBatis-Plus） |
| Developer Tools | Lombok | 自动生成 getter/setter 等样板代码 |

### 步骤 4：生成项目

点击 **Create**，等待 IDEA 下载依赖并生成项目结构。

生成完成后，项目目录结构如下：

```
codenow-backend/
├── src/
│   ├── main/
│   │   ├── java/com/codenow/
│   │   │   └── CodenowBackendApplication.java   # 启动类
│   │   └── resources/
│   │       ├── application.yml                   # 配置文件
│   │       ├── static/                           # 静态资源目录
│   │       └── templates/                        # 模板目录
│   └── test/
│       └── java/com/codenow/
│           └── CodenowBackendApplicationTests.java
├── pom.xml                                       # Maven 依赖配置
└── mvnw / mvnw.cmd                               # Maven Wrapper
```

## 二、验证项目创建成功

在 IDEA 中打开项目后，找到 `CodenowBackendApplication.java`，点击左侧绿色三角形运行按钮，观察控制台输出。

**正常输出示例：**

```
  .   ____          _            __ _ _
 /\\ / ___'_ __ _ _(_)_ __  __ _ \ \ \ \
( ( )\___ | '_ | '_| | '_ \/ _` | \ \ \ \
 \\/  ___)| |_)| | | | | || (_| |  ) ) ) )
  '  |____| .__|_| |_|_| |_\__, | / / / /
 =========|_|==============|___/=/_/_/_/

 :: Spring Boot ::                (v3.x.x)

2026-07-10 ... : Starting CodenowBackendApplication ...
2026-07-10 ... : Tomcat started on port(s): 8080 (http)
2026-07-10 ... : Started CodenowBackendApplication in x.xxx seconds
```

**看到 `Started CodenowBackendApplication` 且端口为 8080 即表示创建成功。**

如果启动报错（如数据库连接失败），属于正常现象，后续配置 application.yml 时会解决。
